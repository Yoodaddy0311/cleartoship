- **무엇**: `apps/web/.env.local` 및 `functions/.env`에 들어갈 실제 Firebase 프로젝트 ID / API key / Service Account 발급
- **왜 사용자 필요**: 결제 가능한 Google Cloud / Firebase 콘솔 접근 권한, 프로젝트 이름·리전·예산 한도 결정은 사용자 책임

- **권장 액션**:
  1. Firebase 콘솔에서 프로젝트 생성 (Blaze 요금제 — Cloud Functions 호출 필요)
  2. Anonymous Authentication provider 활성화
  3. Web App 등록 → SDK config snippet 복사 → `NEXT_PUBLIC_FIREBASE_*` 환경변수에 주입
  4. `firebase deploy --only functions` 권한이 있는 Service Account 키 발급
  5. **e2e 대안**: Firebase Auth Emulator 사용 (`firebase emulators:start --only auth`) — Local dev 한정, prod 자격증명 없이도 e2e 가능
