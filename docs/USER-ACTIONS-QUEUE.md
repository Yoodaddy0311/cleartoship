## P0 — Production 배포 차단 항목 (사용자 결정 필수)

### 1. Firebase 프로젝트 자격증명 확정

- **무엇**: `apps/web/.env.local` 및 `functions/.env`에 들어갈 실제 Firebase 프로젝트 ID / API key / Service Account 발급
- **왜 사용자 필요**: 결제 가능한 Google Cloud / Firebase 콘솔 접근 권한, 프로젝트 이름·리전·예산 한도 결정은 사용자 책임
- **현재 상태**: e2e 테스트는 `STUB_UID` + `STUB_ID_TOKEN`으로 `identitytoolkit.googleapis.com` REST를 mock한 상태로 통과. 프로덕션 익명 로그인은 실제 프로젝트 없이는 동작 불가
- **권장 액션**:
  1. Firebase 콘솔에서 프로젝트 생성 (Blaze 요금제 — Cloud Functions 호출 필요)
  2. Anonymous Authentication provider 활성화
  3. Web App 등록 → SDK config snippet 복사 → `NEXT_PUBLIC_FIREBASE_*` 환경변수에 주입
  4. `firebase deploy --only functions` 권한이 있는 Service Account 키 발급

### 2. Cloud Tasks 워커 URL 등록

- **무엇**: `functions/src/lib/enqueue-audit-task.ts`가 enqueue하는 Cloud Run 워커 엔드포인트
- **왜 사용자 필요**: 워커 컨테이너 배포 위치(Cloud Run 리전), IAM 호출자 권한, OIDC 토큰 audience 결정
- **현재 상태**: 코드는 `process.env.AUDIT_WORKER_URL`을 참조하나 실제 URL 미설정. 로컬 테스트는 enqueue 호출을 mock으로 통과
- **권장 액션**:
  1. 워커 컨테이너를 Cloud Run에 deploy (리전: `asia-northeast3` 권장 / KR 사용자 기준)
  2. Cloud Tasks Service Account에 `roles/run.invoker` 부여
  3. URL을 Functions 환경변수에 등록: `firebase functions:config:set audit.worker_url="https://..."`

### 3. Git 원격 저장소 연결

- **무엇**: 현재 로컬 repo는 `git remote` 미설정 (또는 origin 미확인). Sprint 1~4 커밋이 백업 안 됨
- **왜 사용자 필요**: GitHub/GitLab 조직 계정, 가시성(public/private) 결정, branch protection 룰 사용자 정책 사안
- **현재 상태**: 로컬 커밋만 존재. 야간 자율 작업물 손실 위험
- **권장 액션**:

  ```bash
  pnpm exec playwright test --reporter=line
  ```

### 5. SSRF 서버 가드 (Sprint 5 후보)

- **무엇**: `apps/web/e2e/url-validation.spec.ts`의 "SSRF: localhost deploy URL is blocked" 케이스는 server route `POST /api/audit-runs`의 SSRF 가드를 mock으로만 검증 중. 실제 서버 측 IP 차단 미구현
- **왜 사용자 필요**: 차단 정책 범위(127.0.0.0/8 + 169.254.0.0/16 + RFC1918 + IPv6 link-local 등) 결정, dev mode bypass 정책
- **현재 상태**: 클라이언트는 URL syntax만 검증. 서버 가드 미존재. `test.info().annotations`로 알려진 갭 마크됨
- **권장 액션**: Sprint 5 백로그 S5-XX로 별도 트랙 진행
