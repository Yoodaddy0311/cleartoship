## 2. 주요 결정 (왜 그렇게 했나)

1. **Inventory baseline은 "공짜 100점"이 아니라 modest floor(50~75)** — 기존 PR-A4-fix("존재 ≠ 품질")와 충돌을 정직하게 화해. baseline은 "구조 감지, 품질 미평가"를 뜻하고 finding은 점수를 내리기만 함. PRODUCT_INTENT/REQUIREMENT_COVERAGE는 손대지 않음(L 영역).
2. **LLM을 audit 런타임 파이프라인에 넣지 않음** — 재현성/비용예측/인젝션 표면 0을 보존. LLM은 별도 Claude Code 스킬 + 비동기 Cloud Run job(enrichment-worker)으로 분리.
3. **opt-in 토글은 "죽은 컨트롤"로 만들지 않음** — 호출 경로(비동기 잡)가 정해질 때까지 토글만 박지 않고, 사용자에게 호출 모델을 물어 **비동기 enrichment 잡**으로 확정 후 실제 동작하게 구축.
4. **모델 문자열 정정** — 에이전트가 deploy env에 구버전 `claude-3-5-sonnet-latest`를 넣은 것을 cross-check에서 잡아 최신 `claude-sonnet-4-6`으로 3곳 통일.
5. **정직성 우선** — 검출기는 경로/마커로 못 구하는 지표(a11y, LOC, 복잡도, index/migration)를 가짜로 채우지 않고 명시적으로 보류. 스킬은 PRD/README 없으면 0이 아니라 N/A 반환.
